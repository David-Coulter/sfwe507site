## Executive Summary
The Product Development Management System is an Agile project management tool that allows
users to manage Product Backlog. This system allows team members to add new tasks to the
Product Backlog, set and change priority for a task, move a task from the Product Backlog to the
current Sprint Backlog, and then move a task from the Sprint Backlog to the Ready for Test
status. Once tested, the task is either failed and sent back for re-work in a Sprint or passed and
marked as complete and ready for release.

## Project Development
The release of this software is limited to prototyping in an iterative and incremental
environment. The team will demonstrate partially running components of the software to show
progress and to elicit comments, questions, and change requests after key Sprints.
Please expect and prepare for re-work. Sometimes you will find that what worked before adding
a new feature is broken in the current demo release. This is exactly what happens in professional
software development and is why this makes for a solid learning experience.
We will be gathering and refining requirements, breaking them into manageable tasks, and using
our own system to manage the building of this tool, incrementally, as each piece of functionality
is added.
